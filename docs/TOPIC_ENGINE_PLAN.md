# 选题引擎实施计划

本文档将选题引擎拆成可独立验收的阶段。开发顺序强调先验证运营价值，再逐步自动化。

## Phase T0：领域模型与人工工作流

### 目标

让运营人员可以录入参考案例、创建趋势信号，并获得结构化选题候选。

### 任务

- [x] 定义选题评分领域类型与基础规则；
- [ ] 建立 `trend_signals`；
- [ ] 建立 `reference_contents`；
- [ ] 建立 `viral_patterns`；
- [ ] 建立 `pattern_source_links`；
- [ ] 建立 `cat_content_opportunities`；
- [ ] 建立 `topic_candidates`；
- [ ] 建立 `topic_score_runs`；
- [ ] 支持手工录入参考链接和指标；
- [ ] 支持手工录入节日、天气、内部事件和业务目标；
- [ ] 提供候选创建、编辑、评分和状态流转 API；
- [ ] 保存评分解释、风险和阻断理由；
- [ ] 所有数据写入审计日志。

### 验收

- 能为 10 只测试猫建立内容机会；
- 能手工创建 20 个参考案例和 10 个抽象模式；
- 能生成或编辑选题候选并得到可解释评分；
- 高相似度、高版权风险和高事实风险候选会被阻断。

## Phase T1：案例结构化拆解

### 目标

使用文本或多模态模型，把参考案例拆成传播模式，而不是改写原内容。

### 任务

- [ ] 定义 `ReferenceAnalysis` JSON Schema；
- [ ] 建立案例拆解提示词和版本管理；
- [ ] 分离标题功能、叙事节拍、情绪曲线、视觉语法和互动机制；
- [ ] 支持人工修改和批准分析结果；
- [ ] 将多个案例合并成模式；
- [ ] 标注禁止复用的独特表达、角色和镜头；
- [ ] 默认不把完整第三方作品传给后续生成器；
- [ ] 记录模型、token、费用、耗时和错误。

### 验收

- 同类案例能被归并为稳定模式；
- 生成器只能访问抽象模式和必要摘要；
- 人工能看到模式来自哪些案例，但不能直接一键换皮。

## Phase T2：猫咪机会索引与候选生成

### 目标

把趋势、模式、业务目标和猫咪资产组合成差异化候选。

### 任务

- [ ] 计算猫咪资料完整度；
- [ ] 计算素材可执行性；
- [ ] 计算近期曝光和题材疲劳；
- [ ] 提取猫咪独有事实、视觉特征和新事件；
- [ ] 支持热点驱动、案例迁移、角色驱动、业务驱动和系列驱动；
- [ ] 每次生成 5—10 个差异化候选；
- [ ] 候选必须包含猫咪匹配理由和用户观看理由；
- [ ] 候选必须列出事实来源和素材需求；
- [ ] 支持人工锁定猫咪、平台、目标或模式；
- [ ] 增加同批候选去重。

### 验收

- 每个候选至少包含一个猫咪独有事实；
- 同一批候选在角度、结构或受众上有明显差异；
- 缺少素材或事实时不会通过虚构补齐；
- 推荐结果可以解释“为什么现在讲、为什么是这只猫”。

## Phase T3：原创与风险门禁

### 目标

在生成内容前阻止换皮、事实错误和不可执行选题。

### 任务

- [ ] 标题词面相似度；
- [ ] 文本语义相似度；
- [ ] 故事节拍相似度；
- [ ] 视频镜头结构相似度；
- [ ] 第三方角色、品牌、歌词和台词检测；
- [ ] 猫咪事实来源校验；
- [ ] 内容等级校验；
- [ ] 素材授权和音乐授权校验；
- [ ] 生成原创性说明；
- [ ] 支持 warning 和 blocked 两级风险。

### 验收

- 单一案例逐句换词无法通过；
- 单一视频逐镜头替换猫咪无法通过；
- 组合多个抽象模式并基于真实猫咪资料重构的候选可进入审核；
- 所有阻断都有明确原因。

## Phase T4：图文与视频内容包

### 目标

选题批准后生成可制作的完整内容包，而不是只输出一段文案。

### 小红书图文

- [ ] 标题候选；
- [ ] 封面文案；
- [ ] 6—9 页卡片结构；
- [ ] 每页文案与图片建议；
- [ ] 真实素材匹配；
- [ ] AI 补图提示词；
- [ ] 正文、标签和评论引导。

### 公众号

- [ ] 标题、导语和正文；
- [ ] 事实引用；
- [ ] 图片插入建议；
- [ ] 领养或品牌行动引导。

### 短视频

- [ ] 15、30、60 秒脚本；
- [ ] 前三秒钩子；
- [ ] 分镜、旁白、字幕、音效和节奏；
- [ ] 真实素材、补拍素材和 AI 素材标记；
- [ ] 封面和发布文案；
- [ ] 时长估算和素材缺口检查。

### 验收

- 同一选题的平台版本共享事实和故事核心；
- 运营人员能直接按内容包制作；
- 素材不足会明确提示，而不是静默编造；
- 内容包能追溯角色卡、事实、模式和选题版本。

## Phase T5：发布表现回流

### 目标

使用真实表现逐步修正选题权重，不以主观“像爆款”作为唯一判断。

### 任务

- [ ] 建立 `content_experiments`；
- [ ] 建立 `performance_snapshots`；
- [ ] 支持 CSV 手工导入；
- [ ] 统一不同平台的相对指标；
- [ ] 记录标题、封面、开头和时长实验；
- [ ] 对比预测分数与实际表现；
- [ ] 每周输出高分低表现和低分高表现复盘；
- [ ] 调整权重时保留版本；
- [ ] 数据量足够后再评估学习排序模型。

### 验收

- 每条发布内容能关联到选题和内容包；
- 可以按猫咪、模式、平台、目标和时间段复盘；
- 权重变化有数据依据和回滚能力。

## Phase T6：趋势连接器

### 开始条件

只有人工工作流已证明选题候选有价值，才进入自动采集。

### 任务

- [ ] 评估平台官方接口和授权数据服务；
- [ ] 定义采集频率、保留范围和删除策略；
- [ ] 只保存必要元数据、指标和短摘要；
- [ ] 实现趋势衰减和异常增长检测；
- [ ] 建立来源限流、失败重试和审计；
- [ ] 不绕过登录、验证码、访问控制和平台限制；
- [ ] 为每个连接器记录合规依据。
