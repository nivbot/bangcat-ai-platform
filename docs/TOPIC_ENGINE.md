# 选题引擎技术设计

- 文档状态：V0.1
- 产品优先级：核心能力
- 适用阶段：MVP 到内容增长阶段

## 1. 为什么选题引擎是核心

仅根据猫咪资料直接生成内容，容易稳定地产出“正确但没人看”的介绍文。真正影响传播效果的首要变量通常不是文案润色，而是：

- 此刻讲什么；
- 为什么用户愿意停下来；
- 选择哪只猫承载这个主题；
- 用什么平台形式表达；
- 如何借鉴已验证的传播结构，同时保持原创和事实安全。

因此系统的主链路从“猫咪资料 → 写作”升级为：

```text
趋势与爆款案例
    ↓
结构化拆解
    ↓
可复用模式库
    ↓
猫咪机会匹配
    ↓
原创选题候选
    ↓
评分与风险门禁
    ↓
图文 / 视频内容包
    ↓
人工审核与发布
    ↓
表现数据回流
```

选题引擎不替代猫咪资产库。猫咪资产库提供事实、角色和素材，选题引擎决定这些资产在什么时机、以什么角度进入内容生产。

## 2. “参考爆款”而不是复制爆款

系统需要学习爆款，但不能把复制原文、替换主角或复刻镜头序列当成生产策略。

### 2.1 参考层级

| 层级 | 方法 | 系统策略 |
| --- | --- | --- |
| A | 复制标题、正文、画面或台词 | 禁止 |
| B | 只把原内容中的人物替换成猫 | 高风险，默认禁止 |
| C | 抽象钩子、节奏、情绪曲线和互动机制 | 允许 |
| D | 组合多个模式，并由猫咪真实资料重新构建故事 | 推荐 |
| E | 沉淀帮猫自己的系列栏目和内容格式 | 长期目标 |

### 2.2 允许学习的内容

- 标题功能：悬念、反差、身份错位、结果前置；
- 开头机制：三秒冲突、异常画面、强问题、意外结论；
- 叙事结构：误会—揭晓、失败—升级—回报、观察—证据—结论；
- 情绪曲线：好奇、紧张、幽默、温情、释然；
- 节奏：信息密度、镜头长度、转折位置；
- 互动机制：站队、猜结局、代入、评论补充；
- 平台形式：小红书卡片结构、短视频镜头结构、公众号长故事结构。

### 2.3 不应保存或输出的内容

- 未获授权的完整正文、视频文件和高清素材；
- 可直接还原原作品的完整逐字稿或分镜；
- 原作者独特台词、角色、品牌识别和连续镜头组合；
- 通过轻微改写规避相似度检查的内容。

默认保存案例元数据、指标快照、抽象结构和必要的短摘要，不把第三方完整作品作为生成上下文长期存储。

## 3. 系统边界

### 3.1 输入

1. 猫咪资产：事实、角色卡、照片、视频、领养状态、故事事件；
2. 趋势信号：节日、天气、社会情绪、平台热词、站内事件；
3. 爆款参考：人工录入或合规数据源获得的公开案例；
4. 运营目标：领养、品牌、互动、涨粉、栏目建设；
5. 资源约束：现有素材、可拍摄条件、AI 生成预算、发布时间。

### 3.2 输出

选题引擎只输出可审核的 `TopicCandidate`，不直接发布内容。候选至少包含：

- 一句话选题；
- 用户为什么会看；
- 推荐猫咪及匹配理由；
- 目标平台与内容形式；
- 开头钩子；
- 故事核心与关键转折；
- 事实等级；
- 所需素材；
- 参考的抽象模式；
- 原创性约束；
- 风险和评分解释。

## 4. 核心模块

## 4.1 Signal Collector：信号采集

采集三类信号：

### 平台信号

- 热门主题和内容形式；
- 点赞、评论、收藏、转发、播放等公开指标；
- 发布时间和增长速度；
- 评论区高频问题与情绪。

### 时间信号

- 节日、季节、天气和城市事件；
- 工作日、周末、毕业季、开学季等生活周期；
- 猫咪入住、康复、生日、领养等内部事件。

### 业务信号

- 当前急需曝光的猫；
- 长期等待领养的猫；
- 新增高质量素材；
- 品牌活动、场地合作和公益事件。

MVP 先支持人工录入和 CSV/JSON 导入，不立即实现未经确认的平台自动抓取。

## 4.2 Reference Analyzer：案例拆解

将一个参考案例转为结构化分析，而不是改写全文。

建议字段：

```json
{
  "platform": "xiaohongshu",
  "format": "carousel",
  "audienceTension": "上班疲惫但想获得陪伴",
  "hookFunction": "把猫描述成最稳定的同事",
  "narrativeBeats": [
    "提出异常结论",
    "给出三个生活证据",
    "揭示猫的真实身份",
    "情绪回落到陪伴"
  ],
  "emotionalCurve": ["好奇", "好笑", "认同", "温暖"],
  "visualGrammar": ["大字封面", "一页一个证据", "最后一页行动引导"],
  "interactionMechanic": "邀请用户分享自家猫的职业",
  "restrictedElements": ["原文标题", "原作者固定台词", "原图构图复刻"]
}
```

案例分析必须保留来源 ID 和分析模型版本，但生成器默认只读取抽象后的模式。

## 4.3 Viral Pattern Library：爆款模式库

多个案例可归并为一个可复用模式。

首批模式分类建议：

- 反差身份：猫拥有与外表不符的职业或能力；
- 误会揭晓：前半段像严肃事件，后半段揭示是猫；
- 失败升级：连续失败后出现小回报；
- 观察证据：提出结论，再用三个细节证明；
- 第一人称日记：由猫描述一天或一次冒险；
- 社会情绪映射：把打工、独居、毕业等情绪映射到猫；
- 倒计时或挑战：限定时间完成目标；
- 前后对比：救助前后、入住前后、领养前后；
- 关系反转：本来是人救猫，最后发现猫也在救人；
- 连载悬念：给角色建立持续目标和下一集问题。

模式必须标注适用平台、适用内容等级、所需素材、疲劳度和禁用元素。

## 4.4 Cat Opportunity Index：猫咪内容机会索引

系统不能只挑资料最多的猫。每只猫需要动态计算内容机会：

- 是否可领养及紧迫度；
- 角色辨识度；
- 近期是否有新事件；
- 是否有足够的照片或视频；
- 与当前趋势的语义匹配度；
- 最近曝光频率；
- 已使用题材及重复风险；
- 是否适合真实、改编或虚构内容；
- 人工优先级。

推荐结果必须解释“为什么是这只猫”，不能只给黑盒分数。

## 4.5 Candidate Generator：候选生成

每次任务建议生成 5—10 个差异明显的候选，而不是一次生成一篇稿件。

候选来源应混合：

1. 热点驱动：趋势 × 猫咪；
2. 案例迁移：抽象模式 × 猫咪事实；
3. 角色驱动：角色目标 × 新事件；
4. 业务驱动：领养目标 × 人群情绪；
5. 系列驱动：已有栏目 × 新一集；
6. 反常识驱动：猫咪真实细节 × 意外结论。

为了降低模板感，一条候选至少应组合：

- 一个传播模式；
- 一个猫咪独有事实或视觉特征；
- 一个当下信号或具体人群情绪；
- 一个平台原生表达方式。

## 4.6 Topic Ranker：选题评分

初始评分使用可解释规则，不急于训练模型。

正向维度：

| 维度 | 初始权重 |
| --- | ---: |
| 趋势相关性 | 18 |
| 猫咪角色与事实匹配 | 18 |
| 人类情绪与冲突强度 | 16 |
| 新颖度 | 14 |
| 平台适配 | 12 |
| 素材可执行性 | 8 |
| 领养或品牌价值 | 8 |
| 时效性 | 6 |

扣分维度：

- 与参考案例过度相似；
- 潜在版权或品牌风险；
- 事实不确定或可能误导；
- 同一猫咪、题材或栏目近期疲劳；
- 素材不足却依赖复杂生成；
- 热点与品牌价值冲突。

建议决策：

- `75–100`：推荐进入内容生成；
- `60–74`：人工复核；
- `<60`：暂不生产；
- 相似度、版权风险或事实风险达到硬阈值时直接阻断。

## 4.7 Originality & Rights Gate：原创与权利门禁

内容生成前和生成后都要检查：

- 标题与参考标题的词序和核心表达是否过近；
- 故事节拍是否几乎逐项对应单一来源；
- 视频镜头是否连续复刻；
- 是否出现第三方角色、歌词、台词或品牌元素；
- 是否把热点人物或事件错误嫁接到猫咪；
- 是否使用无授权图片、视频或音乐；
- 是否能说明猫咪独有事实如何改变了内容。

系统应保存“原创性说明”，记录本内容从哪些抽象模式获得启发，以及做了哪些独立创作。

## 4.8 Content Pack Generator：内容包生成

选题通过后，生成器先输出共享故事核心，再输出平台版本。

### 共享故事核心

- 受众；
- 核心冲突；
- 开头钩子；
- 三到五个故事节拍；
- 情绪终点；
- 猫咪事实来源；
- 行动引导；
- 原创性约束。

### 小红书图文

建议输出：

- 3 个原创标题候选；
- 封面文案；
- 6—9 页卡片结构；
- 每页主句、补充文字和图片建议；
- 真实照片优先级；
- 必要时的 AI 配图提示词；
- 正文、标签和评论区引导。

### 公众号图文

建议输出：

- 标题和导语；
- 完整故事结构；
- 小标题；
- 事实引用；
- 图片插入建议；
- 领养或品牌行动引导。

### 抖音 / 视频号

建议输出：

- 15、30 或 60 秒版本；
- 前 1—3 秒冷启动画面；
- 分镜、旁白、字幕和音效建议；
- 每个镜头使用真实素材、补拍素材或 AI 资产的标记；
- 节奏变化点；
- 结尾回报与 CTA；
- 封面和发布文案。

真实猫咪素材优先。AI 图像和视频用于补足无法拍摄的角色化场景，不应取代所有真实证据。

## 5. 数据模型建议

新增表或领域对象：

### `trend_signals`

趋势、节日、天气、内部事件和业务目标，包含有效期、来源、可信度和热度变化。

### `reference_contents`

参考案例元数据、来源平台、发布时间、指标快照、内容格式、短摘要、授权和保留策略。

### `viral_patterns`

抽象后的钩子、叙事节拍、情绪曲线、视觉语法、互动机制、适用范围和疲劳度。

### `pattern_source_links`

模式与多个参考案例的关系，避免把模式误认为单一作品的改写模板。

### `cat_content_opportunities`

猫咪的动态创作机会、素材状态、近期事件、曝光历史和推荐原因。

### `topic_candidates`

选题候选、故事核心、平台、事实等级、匹配模式、所需素材、状态和版本。

### `topic_score_runs`

评分明细、权重版本、风险项、阻断理由和最终排序。

### `content_experiments`

记录同一选题的标题、封面、开头或时长实验。

### `performance_snapshots`

发布后的曝光、停留、完播、互动、关注、详情访问和领养咨询等指标。

### `content_similarity_checks`

与参考案例、历史内容和同批候选的文本、语义、镜头结构相似度。

## 7. API 草案

```text
POST /v1/references/import
POST /v1/references/:id/analyze
GET  /v1/patterns
POST /v1/trends
GET  /v1/cats/:id/opportunities
POST /v1/topic-candidates/generate
POST /v1/topic-candidates/:id/score
POST /v1/topic-candidates/:id/approve
POST /v1/topic-candidates/:id/generate-content-pack
POST /v1/performance/import
```

所有接口都应记录操作人、模型、提示词版本、来源和审计日志。

## 8. MVP 实施范围

第一版不追求自动抓取全网和预测爆款，先验证选题质量。

### 数据范围

- 人工选择 50—100 个公开参考案例；
- 由运营人员记录链接、平台、指标和选择原因；
- 先覆盖小红书图文和 15—30 秒短视频；
- 使用 10 只资料质量不同的猫测试。

### 工作流

1. 人工导入参考链接和摘要；
2. AI 拆解案例为模式；
3. 人工确认或合并模式；
4. 每天生成 5—10 个选题候选；
5. 人工选择 1—2 个；
6. 生成图文内容包和短视频脚本；
7. 人工编辑、制作并发布；
8. 手工或 CSV 回填数据；
9. 每周复盘高分选题与真实表现的偏差。

### 暂不实现

- 绕过平台规则的自动爬虫；
- 自动下载和保存第三方完整作品；
- 无人工确认的热点追逐；
- 自动发布；
- 直接根据少量数据训练“爆款预测模型”；
- 用单一爆款逐句或逐镜头改写。

## 9. 关键指标

### 选题质量

- 候选被人工选中的比例；
- 从候选到可发布内容的平均修改时间；
- 选题重复率；
- 原创或版权风险阻断率；
- 猫咪事实错误率；
- 运营人员对推荐原因的认可度。

### 内容表现

不同平台使用可比的相对指标，而不是只看绝对播放量：

- 首屏停留或前三秒留存；
- 完读率或完播率；
- 收藏、评论、转发率；
- 关注转化；
- 猫咪详情访问；
- 领养咨询；
- 同类账号或账号自身历史基线提升。

### 学习有效性

- 选题预测分数与实际表现的相关性；
- 不同模式、猫咪、平台和目标的表现差异；
- 新模式衰减速度；
- 高分低表现和低分高表现案例的复盘完成率。

## 10. 最关键的技术难点

1. **趋势获取合规性**：优先人工导入、官方接口、获授权服务或公开元数据，不能把绕过限制的抓取作为系统前提；
2. **多模态拆解**：视频需要同时理解画面、字幕、旁白、节奏和评论反馈；
3. **模式抽象质量**：拆得太浅会变成换皮，拆得太深又失去可执行性；
4. **相似度控制**：需要同时检查文字、语义、故事节拍和镜头结构；
5. **猫咪事实稀疏**：素材不足时应降低推荐，而不是让模型编造；
6. **趋势时效性**：趋势和平台形式有明显衰减，需要有效期和速度信号；
7. **归因困难**：选题、标题、封面、制作和账号基础都会影响表现，应通过小规模实验逐步拆分；
8. **冷启动数据不足**：先用人工评分和规则权重，积累真实发布数据后再学习权重。

## 11. 产品原则

- 先选题，后写作；
- 先模式抽象，后原创生成；
- 先真实素材，后 AI 补充；
- 先人工复核，后自动化；
- 评分必须可解释；
- 热点不能覆盖猫咪事实和品牌边界；
- 爆款是结果，不是系统可以承诺的标签；
- 长期壁垒是“猫咪数字资产 × 内容模式库 × 真实表现反馈”。
