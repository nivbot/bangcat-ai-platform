# 帮猫 AI 内容资产库与 MCP 创作系统

本仓库是“帮猫小本本”AI 猫咪内容创作项目的产品、架构与开发协作仓库。

它将作为以下三方之间的长期同步中枢：

- 产品与业务讨论
- ChatGPT 输出的 PRD、方案与迭代决策
- Codex 的代码开发、任务拆解与实施记录

## 项目目标

将现有小程序生产数据库中的猫咪资料，以只读、脱敏、可审计的方式同步到独立的 AI 内容资产库，形成稳定的猫咪数字角色资产，并自动生成适用于公众号、小红书、视频号、抖音等平台的图文和视频创作草稿。

第一阶段采用“自动生成 + 人工审核”的模式，不做自动发布。

长期目标是在 AI 内容资产库外增加 MCP 服务层，在权限、授权和内容安全可控的前提下，让内部 Agent、Codex 或外部创作者调用猫咪角色与创作素材，共同创作“帮猫宇宙”内容，并探索将创作收益反哺流浪猫。

## 当前阶段

当前处于 **MVP 规划与基础架构阶段**。

首个开发目标不是图片生成、视频生成或第三方自动发布，而是先跑通以下闭环：

1. 生产数据库只读同步
2. AI 猫咪资产结构化
3. 猫咪角色卡生成
4. 多平台内容草稿生成
5. 自动事实检查
6. 人工审核与修改
7. 全链路日志和成本记录

## 关键原则

- 生产数据库与 AI 数据库严格隔离
- AI 系统不得直接写入生产数据库
- 外部服务不得直接访问生产数据库
- 真实事实与虚构故事必须明确区分
- 每次生成必须可追溯、可重试、可审计
- 第一阶段优先验证内容质量与业务闭环，不追求全自动化
